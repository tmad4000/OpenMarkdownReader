01

Why another markdown app?

Most markdown apps want you to adopt their system. We think you already have one—it's called the filesystem. OpenMarkdownReader meets you where your files already are.

Open arbitrary files

No vault required. Open any .md, .txt, or .markdown file from anywhere on your Mac. Your Downloads folder, a git repo, a shared drive—it all works.

Local-first, always

Your notes stay on disk where git, Spotlight, backup tools, and your own scripts can reach them. No sync service required, no cloud dependency.

Low-friction editing

Toggle between preview and edit with ⌘E. Auto-save keeps your changes safe. No ceremony, no confirmation dialogs—just write.

Obsidian-compatible

Wikilinks work naturally, so your [[connections]] stay intact. But you're never forced into a vault-first workflow just to read a single note.

02

Built for how you actually work

A focused set of features that make reading and editing markdown feel effortless, without the bloat of a full knowledge management system.

Multi-tab, multi-window

Work on several files at once. Each tab remembers scroll position and edit state.

GitHub-flavored markdown

Tables, task lists, strikethrough, code blocks with syntax highlighting—it all renders.

Folder navigation

Open a folder and browse its contents in the sidebar. Jump between notes without leaving the app.

Drag and drop

Drop a file onto the window to open it. Simple as that.

03

A calm, three-step workflow

  1. 1

    Open

    Press ⌘O or drag a file onto the window. Open a single note or an entire folder. Your recent files are always one click away.

  2. 2

    Read or edit

    Preview mode shows clean, styled markdown. Press ⌘E to switch to edit mode. Changes auto-save as you type.

  3. 3

    Move on

    Close the app. Your file is exactly where you left it—on disk, ready for git, Spotlight, or whatever comes next.

The right tool for loose files
